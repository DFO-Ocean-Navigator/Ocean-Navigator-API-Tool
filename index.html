API Documentation

How To:

The Ocean Navigator uses 2 different methods to receive API queries.
Both methods require the query to be prepended with "http://navigator.oceansdata.ca/"
The first method uses Query Parameters, and the second uses Template Parameters

(1) Query Parameters
The query method uses a base query followed by the input parameters
To indicate the start of variable input, a ? is used.
Similarly, to include more than one variable in the query, seperate each variable with a &
Variables included at the end of a query can be defined using
?variable= ...

Example: 
http://navigator.oceansdata.ca/api/variables/?dataset=giops_day&3d_only


?          : beginning of parameter input
dataset    : parameter with assigned value
=giops_day : assigning value to dataset
&          : indicates next parameter
3d_only    : standalone parameter (no value)
When scripting, longer queries should be written in JSON and converted to encodedURI format

(2) Template Parameters
Template based parameters differ from query based parameters in various ways:
- Parameter Order Matters
- Parameter values are separated by '/'
-Input paramater types must match expected parameter types
This means you cannot provide an iteger if the query is expecting a string.
The expectations for each query are outlined in the documentation below.

Example: 
http://navigator.oceansdata.ca/api/class4/forecasts/class4_20180531_GIOPS_CONCEPTS_2.3_profile


format: /api/class4/<string:q>/<string:class4_id>/


/api/class4/       : Base
<string:q>         : 1st Parameter Value
<string:class4_id> : 2nd Parameter Value
API Queries can be tested by copying and pasting the query into a browser, or can be automated by constructing them in a script
Using JSON to construct queries is very useful for creating long queries.
It is both easier to make changes, as well as easier to read

There are multiple ways to construct an API request in a script. 2 examples are given below.
**Both examples will print 4 lists displaying the depths available for each of the variables specified**

(1) String Replacement
In this example, the 3 curly brace sets "{}" will be replaced by the 3 values given in .format()
These values can be assigned, incremented, or modified to make multiple automated API requests. 
import urllib	

base_url = "http://navigator.oceansdata.ca/api/depth/"
dataset = 'giops_day'
variables = ['votemper', 'vosaline', 'vozocrtx', 'vomecrty']

for variable in variables:
	url = ('{}?dataset={}&variable={}'.format(base_url, dataset, variable))
	f = urllib.urlopen(url)
	output = f.read()
	print(output)

Function Documentation

/api/class4/<string:q>/<string:class4_id>/
example: /api/class4/models/class4_20180512_GIOPS_CONCEPTS_2.3_profile/100

<string:q>         : forecasts (dates) models (?) 
<string:class4_id> : ID of the desired class4 - Can be found using /api/class4/

Returns in-situ observations with global prediction system estimates
/colormaps.png
Returns image of colourmap example configurations
/api/colormaps/
Returns a list of colourmaps
/api/colors/
Returns a list of colours for use in colour maps
/api/depth/?dataset=<string:dataset_id>&variable=<string:variable>
example: /api/depth/?dataset=giops_day&variable=votemper

<dataset_id>  : id of the desired dataset - Can be found using /api/datasets/id
<variable> : Type of data to retrieve - found using /api/variables/?dataset='...'

Returns all depths available for that variable in the dataset
/api/drifters/<string:q>/<string:drifter_id>
example: /api/drifters/vars/300234063261890/

<string:q>          : vars / time (Data Request)
<string:drifter_id> : ID of Drifter of Interest - Options can be found using /api/

Vars - Returns a list of Variables applicable to the specified drifter
Time - Returns the max and min time of the specified drifter
/api/data/<string:dataset>/<string:variable>/<int:time>/<string:Depth>/<string:location>.json'
example: /api/data/giops_day/votemper/823/0/63.51801708389351,-56.13861083984375.json

<string:dataset>  : Dataset id - Can be found using /api/datasets/id
<string:variable> : Type of data to retrieve - found using /api/variables/?dataset='...'
<int:time>        : Time of data collection
<string:Depth>    : Water Depth - found using /api/depth/?dataset='...'
<string:Lat,Lon> : Location of the data you want to retrieve

**All Components Must be Included**

Returns JSON object containing the location, name, units, and value based on the input values
/api/observationvariables/ 
example: /api/observationvariables/

Returns a list of the possible observation variables
/plot/?query=<json:query> 
example: Click for Example

<json:query> = {
	dataset   : Dataset id - found using /api/datasets/id
	names     :
	plottitle : Title of Plot (Default if blank)
	quantum   : (year, month, day, hour)
	showmap   : Include a map of the plots location on the map
	station   : Coordinates of the point/line/area/etc
	time      : Time retrieved data was gathered/modeled
	type      : Plot Type (Check Navigator for Possible options)
  variable  : Type of data to plot - Options found using /api/variables/?dataset='...'
}

Plot Types - map, transect, timeseries, ts, sound, profile, hovmoller, observation, drifter, class4, stick

**Query must be written in JSON and converted to encodedURI**
**Not all components of query are required

This query is used to create plots of all types, as well as to create some file types including csv
/api/<string:q>/ 
example: /api/lines/

<string:q> : Zone Type Can be (points,lines, areas, or class4)

Returns predefined  points / lines / areas / class4's
/api/datasets/ 
Will return a list of possible datasets and their corresponding data
/api/datasets/id 
Will return a list of Dataset names and their corresponding id's
/api/<string:q>/<string:projection>/<int:resolution>/<string:extent>/<string:file_id>.json 
example: /api/class4/EPSG:3857/9784/-15618972,1596939,4487023,12369057/class4_20180510_GIOPS_CONCEPTS_2.3_profile.json
--At this time, only the class4 option is available

<string:q>          : Type of data (points, lines, areas, class4, drifters, observations)
<string:projection> : Current projection of the map (EPSG:3857, EPSG:32661, EPSG:3031)
<int:resolution>    : Current zoom level of the map
<string:extent>     : The current bounds of the map view
<string:file_id>    : Class 4 file ID

**All Components Must be Included**
**Used Primarily by WebPage**
At this time will return a list of class4's and their data
/api/<string:q>/<string:q_id>.json' 
example: /api/drifters/meta.json

<string:q>    : Type of Data (areas, class4, drifters, observation)
<string:q_id> : metafile name drifters: meta, observation: meta

Returns a list of all the objects of the selected type
/scale/<string:dataset>/<string:variable>/<string:scale_min, scale_max>.png 
example: /scale/giops_day/votemper/-5,30.png

<string:dataset>  							: Dataset id - found using /api/datasets/id
<string:variable> 							: Type of data to retrieve - found using /api/variables/?dataset='...'
<string:scale_min,scale_max>    : Desired Scale

Returns a coloured scale bar using the provided scale
/stats/?query=<string:query> 
example: /api/drifters/meta.json

<string:query> = {
  dataset  : Dataset id - found using /api/datasets/id
  variable : Type of data to plot - Options found using /api/variables/?dataset='...'
  time     : Time retrieved data was observed / modeled
  depth    : Water Depth - found using /api/depth/?dataset='...'
  area     : Selected Area
}

**Query must be written in JSON and converted to encodedURI**
**Not all components of query are required

Returns a list of statistical variables associated with the plot selected.
This may include variable name, units, min, max, mean, median, and stddev
/subset/?query='...' 
example: Click for Example

query = {
	Output Format : File Format
	Dataset id    : id of the desired dataset - found using /api/datasets/id
	Variables     : data type ex. temperature - found using /api/variables/?dataset='...'
	Min range     : Minimum of bounding box - displayed below
	Max range     : Maximum of bounding box - displayed below
	time          : Data Time Period
	zip           : Takes a value of 1 or 0 [1: create a zip file, 0: do not zip file]
}

Bounding Box:


**Query must be written in JSON and converted to encodedURI**
**Not all components of query are required
Returns a subset of data based on the criteria provided, this can be in multiple different file formats
/api/timestamps/?dataset=<string:dataset> 
example: /api/timestamps/?dataset=giops_day

<string:dataset> : Dataset id - Can be found using /api/datasets

Finds all timestamps associated with a dataset
/api/timestamp/<string:old_dataset>/<int:date>/<string:new_dataset> 
example: /api/timestamp/giops_day/822/riops

<string:old_dataset> : Previous dataset used
<int:date>           : Date of desired data - Can be found using /api/timestamps/?datasets='...'
<string:new_dataset> : Dataset to extract data - Can be found using /api/datasets

**Used when Changing datasets**
/api/variables/?dataset=<dataset_id>&3d_only&vectors_only&vectors 
example: /api/variables/?dataset=giops_day

**Only use variables required for your specific request**

dataset_id   : Dataset to extract data - Can be found using /api/datasets/id
3d_only      : When included, ONLY variables with depth will be shown
vectors_only : When included, ONLY variables with magnitude will be shown 
vectors      : When included, magnitude components will be included

**Boolean: True / False**
Will return a list of variables in the specified dataset, this output can be filtered by including keywords in the query
